# Engineering Rules

Sempre:

- Investigar antes de implementar
- Seguir padrões existentes
- Não criar arquitetura nova sem necessidade
- TypeScript strict
- Evitar any
- Evitar duplicação
- Código pronto para produção

Fluxo obrigatório:

investigator
→ planner
→ implementer
→ reviewer
→ tester

Toda task deve seguir esse fluxo.
